pi extension

pi-fusion

A compound-AI planning pass for pi. Several read-only model calls explore your task in parallel, then merge into one actor turn. 

pi install git:github.com/leblancfg/pi-fusion
GitHub README Preset docs pi packages

How it works

What it does

Read-only workers plan in parallel, then their notes are injected into the actor's system prompt for that turn only.

Why bother

A cheap test-time compute fanout can catch missed files, alternate fixes, or risks before the actor starts editing.

When to use it

Fuzzy bugs, unfamiliar code, refactors, reviews. Turn it off for tiny edits where latency costs more than the plan.

Get started

Open settings

/fusion

Set worker count, pick models, and save/load your own presets from the TUI pane.

Run locally

pi -e ./extensions/pi-fusion/index.ts

Test a checkout before installing it as a package.

Check the repo

pnpm run check

Runs format, lint, typecheck, tests, and the smoke test.
